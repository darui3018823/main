# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

Personal website for [daruks.com](https://daruks.com/), deployed via GitHub Pages from `main`. Mostly static HTML/CSS/JS, except the contact page which is a Vite + React SPA. Content is in Japanese.

## Commands

```sh
pnpm install            # install all deps (pnpm workspace: root + contact-app)
pnpm css build          # rebuild dist/output.css from input.css (Tailwind v4)
pnpm lint               # ESLint (flat config in eslint.config.mjs)
pnpm contact:dev        # Vite dev server for contact-app
pnpm contact:build      # build contact-app → contact/ (commit the output!)
go run main.go          # local static file server at http://localhost:8080
```

There are no tests (`pnpm test` is a stub).

## Build outputs are committed — keep them fresh

GitHub Pages serves the repo as-is, so build artifacts are checked in. CI (`.github/workflows/ci.yml`) fails the PR if either is stale:

- `dist/output.css` — regenerate with `pnpm css build` after editing `input.css` or adding/removing Tailwind utility classes in any HTML/JS.
- `contact/` — **never edit by hand.** It is wiped and regenerated by `pnpm contact:build` from `contact-app/` sources. Run the build and commit the result alongside source changes.

CI runs three jobs on PRs and pushes to `main`: ESLint, Tailwind build + `dist/output.css` freshness, contact-app Vite build + `contact/` freshness.

## Architecture

- **Root pages (static)**: `index.html` + `script.js` + `styles.css` form the home page — Liquid Glass hero cards, a profile card with Discord presence (Lanyard API polled every 30s, avatar synced from Discord CDN), paginated link tiles, and a slide-out side-panel nav. `projects/` has its own `projects_styles.css` / `projects_script.js`. Styling is a mix of hand-written CSS (`styles.css`, custom classes like `liquid-glass`, `side-panel`) and Tailwind utilities compiled to `dist/output.css`.
- **Contact SPA**: sources in `contact-app/` (React 19, Vite 7, pnpm workspace member). Built with `base: '/contact/'` into `contact/`. Category selection is driven by the `?c=` query param. The vite config's `copyContactEntryToNestedRoutes` plugin also emits `contact/privacy-policy/index.html` (the privacy policy lives inside the SPA; the old `legal/privacy-policy/` static page is gone).
- **Local dev**: `main.go` is a dev-only static server on `:8080` (not deployed; excluded from CI). The Vite dev server proxies shared root assets (`/styles.css`, `/script.js`, `/dist`, `/images`) to it, so run both when developing contact-app against the real site chrome.
- `_remove/` is scrap/parking space and is excluded from lint.
